# 边集生成优化说明

## 背景

抽象地图生成需要在已有分层点集上生成边集，并满足以下目标：

- 图整体连通，可用于路径规划。
- 边尽量符合平面图约束，避免地图道路出现大量交叉。
- 边的类型和权重能体现点集层级，例如普通局部边、区域主干边、核心点高层连接边。
- 能支撑 `10000+` 节点规模，避免全量点对带来的平方级开销。

旧实现采用全量 MST 保证连通，再枚举候选边并逐条检查是否与已有边相交。该方法在小规模数据下可行，但在 `10000+` 节点时主要瓶颈包括：

- 全量 Prim MST 需要反复扫描所有未选节点，接近 `O(n^2)`。
- 局部候选和兜底候选存在全量点对枚举，接近 `O(n^2)`。
- 补边时每条候选边扫描已有边做相交检测，接近 `O(m^2)`。

## 当前策略

当前实现改为局部平面构图：

1. 使用空间网格索引点集。
2. 在局部半径内生成近邻候选边。
3. 使用 Gabriel 图空圆条件筛选候选边。
4. 在 Gabriel 候选边上使用 Kruskal 算法生成连通骨架。
5. 按层级优先级补充 Gabriel 候选边。
6. 若目标边数仍不足，再使用局部短边候选补边。
7. 补边交叉检测使用边空间索引，只检查候选边附近网格内的既有边。

## 关键算法

### 空间网格近邻查询

点集由泊松磁盘采样生成，局部密度相对稳定，适合用规则网格索引。

网格用于：

- 查询某个节点局部半径内的其他节点。
- 判断 Gabriel 空圆内是否存在第三个点。
- 避免全量点对枚举。

候选边不再来自所有 `(i, j)` 点对，而是来自每个点附近固定半径内的邻居。对于分布较均匀的点集，平均邻居数接近常数，因此候选生成接近线性。

### Gabriel 图候选

对于两个点 `u` 和 `v`，如果以 `uv` 为直径的圆内不存在其他点，则边 `(u, v)` 是 Gabriel 边。

Gabriel 图具有两个重要性质：

- 它是 Delaunay 三角剖分的子图。
- 它是平面图，边之间不会产生交叉。

因此当前实现不直接引入第三方 Delaunay 三角剖分库，而是使用更轻量的 Gabriel 空圆判定生成平面候选边。

判定流程：

1. 计算 `u` 和 `v` 的中点。
2. 以 `|uv| / 2` 为查询半径。
3. 使用空间网格查找圆附近节点。
4. 若不存在落入该圆内部的第三点，则接受该候选边。

### Kruskal 连通骨架

旧实现使用全量 Prim MST。新实现先构造 Gabriel 候选边，再在候选边集合上运行 Kruskal：

1. 将 Gabriel 候选边按长度排序。
2. 使用并查集维护连通分量。
3. 依次加入不会形成环的边。
4. 当只剩一个连通分量时停止。

由于欧氏 MST 是 Delaunay 图的子图，而 Gabriel 图也能覆盖大量局部短边，对于当前泊松点集场景，Kruskal 在 Gabriel 候选上可以高效生成连通骨架。若初始查询半径不足以连通，生成器会逐步扩大局部查询半径重建候选。

### 层级优先补边

点集已经维护 `address` 层级地址。边生成继续使用层级信息：

- `commonAddressLevel(a, b)` 表示两个点共享的最高层级区域。
- 同区域边优先级高于跨区域普通边。
- 核心点之间的高层连接优先级更高。
- 边容量按共同层级放大。

候选边排序规则：

1. 优先级低的数值先加入。
2. 共同层级更高的边先加入。
3. 边长更短的边先加入。

这样可以在保持平面图的同时，让边集更接近抽象地图结构：底层局部连通，高层核心点形成更强连接。

### 局部短边填充

Gabriel 图偏稀疏。当目标边数较高时，仅依赖 Gabriel 候选可能无法补满。

当前实现会额外生成局部短边候选：

- 仍然只查询空间网格内的近邻。
- 不做全量点对枚举。
- 补边前检查是否与已有边相交。

这一步用于在不破坏平面性的前提下提高边数，适合抽象地图中道路密度较高的区域。

### 边空间索引

旧补边策略中，每条候选边都要扫描全部已有边判断是否相交，边数较大时非常慢。

当前实现新增边空间索引：

1. 将已加入的边按包围盒覆盖的网格单元登记。
2. 检查新候选边时，只遍历该候选边包围盒覆盖网格中的已有边。
3. 忽略共享端点的边。
4. 对局部候选边和附近已有边做线段相交检测。

这样将补边阶段从全图扫描变为局部扫描，在大规模点集上速度提升明显。

## 复杂度对比

| 阶段 | 旧策略 | 当前策略 |
| --- | --- | --- |
| 连通骨架 | 全量 Prim，约 `O(n^2)` | Gabriel 候选 + Kruskal，约 `O(k log k)` |
| 候选生成 | 全量点对，约 `O(n^2)` | 空间网格局部近邻，接近 `O(n)` |
| 平面性检查 | 候选边扫描所有已有边，最坏约 `O(m^2)` | 边空间索引局部检查 |
| 层级优先 | 支持 | 支持 |
| 大图适配 | 较差 | 较好 |

其中 `n` 是节点数，`m` 是已生成边数，`k` 是局部候选边数量。对于泊松点集，`k` 通常接近 `c * n`。

## 取舍与后续方向

当前方案优点：

- 不引入第三方几何库。
- 对泊松点集表现稳定。
- 生成结果保持平面图特征。
- 层级边优先级仍保留。
- 能支撑 `10000+` 节点规模。

当前方案限制：

- Gabriel 图比完整 Delaunay 图稀疏，因此高目标边数需要局部短边填充。
- 边空间索引使用边包围盒覆盖网格，可能检查少量无关边，但比全图扫描快很多。
- 大图没有默认执行精确全量交叉验证。

后续可以考虑：

- 引入稳定的 Delaunay 三角剖分实现，直接以三角剖分边作为候选集。
- 为边空间索引使用更精细的线段穿格算法，减少包围盒带来的多余检查。
- 增加分层边类型标签，例如 `local`、`regional`、`core`、`cross_region`。
- 为大图增加抽样式或空间索引式平面性验证工具。
- 根据不同层级设置不同搜索半径，让核心点连接更长、普通点连接更局部。
